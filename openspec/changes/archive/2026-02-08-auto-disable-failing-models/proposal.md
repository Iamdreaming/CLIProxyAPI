## Why

当某个供应商的特定模型短时间内多次调用失败时，系统应自动将该模型禁用一段时间，避免持续向已知有问题的模型发送请求，提高系统的健壮性和用户体验。这类似于熔断器(Circuit Breaker)模式在模型级别的实现。

## What Changes

- 新增供应商模型级别的自动禁用机制
- 在配置中支持配置自动禁用的阈值参数（失败次数、时间窗口、禁用时长）
- 在管理 API 中新增查看被自动禁用的模型列表和手动恢复接口
- 禁用状态随请求失败自动累计，禁用时间结束后自动恢复可用
- 与现有的供应商级别启用/禁用机制共存

## Capabilities

### New Capabilities
- `model-failure-auto-disable`: 供应商模型级别的自动禁用机制
  - 跟踪模型请求失败次数
  - 在滑动时间窗口内超过阈值时自动禁用
  - 禁用期间跳过该模型的选择
  - 支持配置失败阈值、时间窗口、禁用时长
  - 管理 API 支持查询/手动恢复被禁用的模型

### Modified Capabilities
- `vendor-enable-disable`: 需要扩展以支持模型级别的 enabled 状态，而不仅仅是供应商级别

## Impact

- **配置系统**: 需要在供应商配置中增加自动禁用相关的配置字段
- **模型注册/路由**: 模型选择逻辑需要检查模型级别的 enabled 状态
- **执行器**: 所有执行器需要在请求失败时通知自动禁用组件
- **管理 API**: 新增端点用于查询和操作模型禁用状态
- **存储**: 如果需要持久化禁用状态，需要扩展存储模型
